Abstract

The core module of the PreMOn ontology extends lemon (Lexical Model for Ontologies) and defines the main abstractions for modelling predicates with their arguments, selectional constraints, predicate and argument relations and alignments between different predicate models.

Introduction

The core module of the PreMOn ontology is an extension of lemon (Lexical Model for Ontologies) for representing predicate models. The following features of lemon are exploited: * modelling of lexical entries (class LexicalEntry) and their associated forms; * linking of lexical entries to the lexical concept (class LexicalConcept) denoting their intensional (informal) meaning; * representation of subcategorization frames (e.g., VerbNet frames) through classes SyntacticFrame and SyntacticArgument; * linking of lexical entries, lexical concepts and arguments of syntactic frames to corresponding ontology entities (classes, properties, individuals). The core module of the PreMOn ontology extends lemon with the following concepts: * classes Predicate, SemanticArgument and associated properties to model predicates with their semantic arguments, the latter being defined locally to a predicate; * class SemanticRole and associated properties to model the role an argument may play, defined globally (e.g., thematic roles of VerbNet, role enumeration of PropBank and NomBank); * class SemanticType and associated properties to model selectional constraints and other semantic restrictions in predicate models * class Conceptualization reifying the evokes relation, on top of which SKOS closeMatch and exactMatch alignment links between different predicate models can be defined; * classes Example and Markable and associated properties (taken from NIF) are used to represent predicate usage examples.
